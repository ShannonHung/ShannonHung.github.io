Spark UI Monitoring and Debugging

Introduction

To change Spark’s log level, simply run the following command:

  • Job: Spark Jobs
  • Stages:individual stages or relevant tasks
  • storage: info and data cached in application
  • environment: info about configuration and setting of application
  • SQL: structured api queries (SQL, dataframes)
  • Executors: detailed info about each executor running our app
  1. 一個 task 有多個 stages，一個 stage 有多個 tasks。
    1. 第一階段的8個任務: 這說明了Spark如何處理可分割的CSV文件。它將文件分割成多個部分，並在機器上不同的核心之間均勻分配工作。
    2. 第二階段的2個任務: 這個階段只有兩個任務，因為進行了一次重新分區（repartition）操作，將數據移動到兩個分區中。
    3. 最後階段的200個任務: 這是由於Spark的默認shuffle分區數量為200。Shuffle是Spark中數據重新分佈的一個過程，用於在不同的階段間傳輸數據。

第二部分的图表，显示了触发action的job名字，它通常是某个count,collect等操作。有spark基础的人都应该知道，在spark中rdd的计算分为两类，一类是transform转换操作，一类是action操作，只有action操作才会触发真正的rdd计算。
