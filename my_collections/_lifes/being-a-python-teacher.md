# 契機
一開始，從實驗室的學長得知他們有在鬥陣教書，但是因為課業忙碌的關係，所以在因緣際會下，我嘗試去接手他們的工作。這也是我第一次正式冠上講師的名稱去教書。

大概從六月的時候開始準備教材，那時我想說一定要告訴小學員我的心路歷程!當初在想的時候想著，我上完大學到目前為止覺得必學的一些基本知識。
其中我覺得最棒的是版本控制! 可惜沒有太多時間教到，如果好奇教材我仍會放在下面給各位做參考。
接下來我會依序介紹我的教學大綱, 主要參考資料, 使用教材及心得與學員的回饋。



# 課前調查
在設計教材大綱之前我就先製作一個表單，大概了解一下小學員來上課的真正原因與目的。以此做為教材的根基，若是他們想要把基礎程式打穩，我就教得更仔細，如果想要趕緊有甚麼樣的作品或是產出我就會選擇教快一點等等，或是有的小學員已經對未來有所規劃，在未來打算申請該科系或是想當工程師，我就會針對未來的相關職業多做一點介紹。

在透過google form表單調查完之後發現:
> Q1這個會決定我要參雜多少遊戲成分在內

Q1: 你是國中生還是高中生? (暑假後成為高中生，請填高中生)
A1: 大概有95%的小學員都是高中生

> Q2這個會決定我教學的速度

Q2: 以前有寫過程式的經驗嗎? (如果「沒有」請跳過 問題3.)
A1: 大概有65%的小學員寫過程式，像是Python, scratch, C++等等

> Q3這個讓我思考我該如何讓他們產生更多興趣

Q3: 最重要的問題，為甚麼會想要參加程式體驗營呢? 
A3: 大概有75%的人認為高中或大學會用到，所以才來。 再來是高達60%的小學員想要找到興趣。再者是想要自我成長。
![Survey](/assets/images/python_teaching/research/Q4.png)

> Q4這個讓我了解他們為何而來的目的

Q4: 你主要希望從課堂中有甚麼收穫 ?
A5: 希望上完課，了解程式的基本知識大概是90%。希望培養寫程式的思維，以後就能夠自由的寫出程式了高達75%。
![Survey](/assets/images/python_teaching/research/Q5.png)

> Q5這個讓我思考該帶領他們做出自己的第一份程式

Q5: 你比較喜歡哪一種小作品?
A5: 大概80%的小學員想要寫生活上實際能看到的程式。

# 課程準備
## 使用工具
1. **Python基礎的依據**：「Python 技術者們：實踐！ 帶你一步一腳印由初學到精通」
2. **Git教材製作的來源**： 為你自己學Git  https://gitbook.tw/ 
3. **繳交作業**：Google Classroom
4. **上課工具**：Google Meet, Kahoot
5. **上課協助工具**：PowerPoint, Ipad Pro, Goodnotes, Ipad Airplay(投影ipad上電腦)

### 視訊工具
因為疫情關係，所以無法去北商進行實體授課，因此我一直在尋找甚麼樣的工具比較好，我是用了Teams, Google Meet, Zoom等視訊工具。而工具需要基本達到以下功能:
1. 能夠分組討論
2. 能夠分享畫面
3. 網路連線品質穩定
4. 小學員要能夠方便的使用

最後選擇Google Meet，因為除了能達到以上功能外，也能在小組討論開啟小白板共同編輯，有助於團隊間的互動，還有小學員只要有瀏覽器就能使用的方便性。

### 上課協助工具-投影ipad畫面
因為遠端最困難的就是無法使用白板，因此我得思考我要怎麼比較方便的在電腦上面進行手寫或是畫圖，來進行教學。因為大學平時有在使用ipad做筆記的習慣，所以我的目標就是找到一個可以把ipad畫面投影到window畫面的工具，剛好在github上面找到原始碼，然後可以在window系統與ios系統之間投影做串接的工具，我就可以透過分享螢幕的方式來進行授課。

下載資源: https://github.com/xfirefly/Airplay-SDK

### 上課方式
1. 問小學員，如果沒有問題的請用google meet功能舉手讓我知道
2. 待人數累積過半或是3/2我會進行下一個步驟
3. 如果有人沒有跟上，我會把每次教完一個小part就上傳剛剛上課進度到github上面並且提供大家連結去做一個參考

# 課程規劃 Day01
收集完表單資料後，我就開始規劃我的課程大綱，並且放入我想要的課程內容，以及我最希望的就是，當你在選擇寫程式當工程師這條路，或是在大學選擇科系的時候，能夠有自信的、開心的，說出你不後悔走這條路。

因此在第一天我添加一個契機，也就是漸漸地帶領他們了解程式的世界掌甚麼樣子，那平時如果沒有老師可以如何培養自學能力? 寫程式最重要的是甚麼? 
1. 為甚麼寫程式? 
2. 怎麼開始學習寫程式?
3. 產品開發會有那些角色?

在帶領大家了解一些概念以及建立基礎之後，我將開始帶小學員安裝環境並且正式介紹python的一些基本知識與型態，然後漸漸地帶小學員知道如何跟電腦溝通(也就是使用input跟output)。因為很多人第一次接觸程式，所以我大概教到if else的程度就可以點到為止了。從if else開始我會**帶入BMI的例子作為介紹，其中主要也是想教大家寫程式就是一步步的累積最終完成成品**
1. Python環境安裝
2. Python基本介紹與基本型態: 何謂變數
   1. 資料型別的種類
   2. 何謂變數?
   3. 變數命名規則?
   4. 運算式與符號及優先順序
   5. Python內建的函式
3. Python的資料結構
   1. 使用資料結構的理由?
   2. List? Tuple? Set? Tuple?
   3. 資料結構的常用函式?
   4. 容器相關的方法?
4. Python的流程控制
   1. 文字輸入與輸出的技巧
   2. if 判斷式

## 回家功課
當然少不了回家功課囉! 但是一開始也不會太難。
1. 主要是請小學員把今天上課的結合input, output, if else的程式碼進行整理並且再每一行加上註解，同時介紹註解的重要性存在。然後交上去Google Classroom。


# 課程規劃Day02
隔天的早上9:00開始上課，到下午五點，因此早上的開頭為了讓大家回想起昨天的課程內容，所以我製作了Kahoot小遊戲，讓大家醒醒腦! 為了增加一點趣味性，所以我讓第一名的贏家可以少一題今天的作業內容!! (作業也可以從上面Download Material中找到喔!)

**第二天主要內容如下:**
1. 複習昨日上課內容
2. Python的流程控制
   1. While迴圈
   2. For迴圈
   3. Exception例外處理

第二天的重頭戲主要是while and for。所以教得比較久，在教之前我先介紹一下何謂while跟for，以及他們兩者之間的差別跟使用情況等等。然後結合上課邊打程式(程式的範例也可以在Download Material中找到喔)來讓他們漸漸了解使用情況。並讓他們自己打過一遍，沒問題就可以舉手，那我就會進行下一個程式介紹。確保讓大家跟得上腳步。

## while迴圈
我就會透過BMI來帶大家嘗試一下，如果使用者輸入關鍵字Q或是q就可以停止進行輸入的樣子，讓小學員了解一下while其實很適合用在沒有次數限制的情況下。

## For迴圈
我就會帶大家了解，如何利用前面所教的容器把所有人的BMI存進去，並且利用for迴圈進行一個計算平均或是找最大最小的動作。

## Exception
因為while for迴圈佔了一整天的時間，所以我只先教簡單的部分，也就是try ...except...如何使用，更詳細的保留到第三天再來進行。

## 回家功課
因為第二天就交到for跟while所以功課可以有更多的變化，所以我就出的作業比較多，但也不會讓他們就直接空空的從零開始寫，我會先把程式先寫好，然後挖空並加上`TODO1: 請幫我加上if判斷月份...`等提示字樣，讓他們嘗試自己幫我把程式碼完成。

以及嘗試錯誤，並且幫我找到以下五種Exception! 我再上課的時候很鼓勵小學員盡情的嘗試各種錯誤與極限，我希望能讓他們知道，錯誤或失敗並不可恥，可恥的是知錯卻當作沒看到。
   1. NameError
   2. TypeError
   3. ZeroDivisionError
   4. IndexError

# 總結
我覺得這次的教學經驗真的很值得! 我的收穫如下:
